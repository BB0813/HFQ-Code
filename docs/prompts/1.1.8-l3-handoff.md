## 2. 后端 L3（核心）

### 2.1 Opt-in

- `silentInstall===true` 仅当用户曾确认；写入 `silentInstallAcceptedAt`  
- IPC `setPrefs` 若直接写 `silentInstall:true` 而无 AcceptedAt，main 可拒绝或补写时间戳（与 FE 约定一种，写进 FRONTEND-IPC）  

### 2.2 Schedule + 静默安装

实现路径（名称可调，行为必须）：

1. **仅 NSIS 包**（文件名/asset 判别；portable 拒绝 silent）  
2. 校验 `filePath` ∈ `userData/updates`、`.exe`、可读  
3. 若存在 `SHA256SUMS` 或 check 返回的摘要则校验；否则 warning 仍允许（产品可后续收紧）  
4. 写 `pending-install.json`：`{ version, filePath, mode:"silent", scheduledAt }`  
5. **`app.quit()`**（先尽量结束 session worker；running turn 时 FE 应先提示）  
6. **detached spawn** 安装包，参数以 **真实 electron-builder NSIS 实测** 为准（常见 `/S`；`perMachine` 可能触发 **UAC** — 允许）  
7. 禁止再走「只 openPath 打开向导」作为 L3 主路径（L2 保留为回退）  

参考入口：扩展 `update:install` 增加 `mode: "silent" | "ui"`，或新 `update:installSilent`；**silent 要求 prefs.silentInstall**。

### 2.3 启动恢复

新版本启动：

- 读 pending marker  
- 成功（当前 version ≥ target）：清 marker + 可选 `update:state`/`session` 级通知「已更新到 x」  
- 失败/残留：Settings 可「重试静默安装」或「打开安装向导」(L2)  

### 2.4 测试

- pending marker 读写 / 路径拒绝  
- silentInstall=false 时 silent API 拒绝  
- spawn 用 mock child_process（不真装）  
- 手工：`pack:win` 旧版装新包 L3 一条（交付中写步骤）
