## 2. 后端（main + config）

### 2.1 `prefs.updatePolicy`

在 `@hfq/config` schema/store/`withPrefs` 增加（全 optional，旧配置不炸）：

```ts
updatePolicy?: {
  autoCheck?: boolean;           // default true
  autoDownload?: boolean;        // default false
  checkIntervalHours?: number;   // default 24, clamp e.g. 1..168
  silentInstall?: boolean;       // default false · 1.1.7 存储即可，行为禁止自动装
  silentInstallAcceptedAt?: string | null;
}
```

- `setPrefs` / `getConfig` 透传  
- 与现有 `checkUpdatesOnStartup` 协调：startup check 仍尊重旧字段；`autoCheck===false` 则不后台周期检查  

### 2.2 L1 自动下载

在 `apps/desktop/electron/main.cjs`（及 update-download 若需）：

1. 启动 check 完成后：若  
   `updatePolicy.autoDownload === true`  
   且 `updateAvailable`  
   且本地无对应版本完整 `.exe`  
   → 调用现有 downloader.start（recommended NSIS asset，非 portable）  
2. 周期：`checkIntervalHours`（结合现 6h throttle，避免刷爆 API）  
3. 失败：不抛到崩溃；状态 `failed` + 可查询 error  
4. **禁止**因 autoDownload 自动 `install` / `openPath` / quit  

### 2.3 状态可观察

扩展 `getUpdateDownloadStatus` 或新增轻量字段，使 UI 能区分：

- `idle | checking | downloading | ready | failed | cancelled | up_to_date`  
- `availableVersion?`, `currentVersion`, `filePath?`, `percent?`, `error?`, `autoDownloadEnabled?`

若改动协议，**一行**写入 `docs/FRONTEND-IPC.md`。

### 2.4 L2 安装路径

- 保持 `update:install` 确认 + `shell.openPath`  
- 确保 `resolveInstallerPath` 在 autoDownload 完成后能命中文件  
- `quitSuggested` 保留  

### 2.5 测试

- config：updatePolicy merge / clamp  
- 若可单测 downloader：autoDownload 条件分支（mock check 结果）  
- 不要求真网下完整包
