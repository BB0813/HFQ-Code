## 4. 接缝结论

### 双路径
- create/open 的 `onEvent` 均 `upsertGoalFromEvent`（local + worker 共用 SessionManager）  
- compression / compactMaxChars / coding profile 仍经 sessionPrefs → create/open  
- goals merge 在 `AgentSession.init` resume 路径，cold open 测覆盖  

### 可观察性
- LLM compact：provider messages 内 system note + UI 可见 `message.completed` role system  
- 启发式 compact：仅压缩 messages，不强制刷 UI note（与 1.1.5 一致；可接受）

### 已知非阻塞
1. **Memory index P2** — 未做；search 仍 BM25/JSON  
2. **孤儿 goal** — 本轮 FE 已修；无自动化 UI 测  
3. **`void upsertGoalFromEvent`** — best-effort 异步；测里 `setTimeout(50)` 已覆盖常见竞态  
4. **工作区大量未提交 diff** — FE polish + BE 1.1.6 同树；发版前需一次 commit/release train  
5. **package 版本仍 1.1.5** — 正确；发版 Agent 再 bump

## 5. 建议手动冒烟（产品侧 5 分钟）

1. **Settings**  
   - 填压缩 model id + `compactMaxChars=8000` → 保存  
   - 换 Coding Profile → Header chip 立即变；toast 提「新会话生效」  
2. **长对话 / `/compact`**  
   - 多轮灌上下文后发送 → 若配置了 compression，Chat 出现 `[context compacted · llm]`  
   - 清空压缩 model → 仅启发式、无 llm note  
3. **`/goal 梳理 workspace 结构`**  
   - Tasks「目标任务」有行；banner 显示 objective  
   - 杀进程 / 重开同一 session → goal 仍在  
   - 删除会话 → `%data%/sessions/<id>.goals.json` 消失  
4. **文档**  
   - workspace 放 `.docx` / `.md` → 让 agent 用 `read_document`（或提示打开文档）  
5. **Mermaid**  
   - 流式输出未闭合 fence 不崩；完整 ````mermaid` 块结束后出图
