## 2. 必须完成（按优先级）

### P1 · Terminal 重连（B1-2）— **核心缺口**

路径：

- `apps/desktop/renderer/src/features/terminal/TerminalPanel.tsx`
- `apps/desktop/renderer/src/store/terminal-store.ts`
- `apps/desktop/renderer/src/lib/hfq.ts`（补类型）

#### 问题

当前：单 xterm + 只写 **active** 的 `onPtyData`；切路由 remount 后 **主进程 PTY 仍活着，但 xterm 空**；后台会话输出在 inactive 时丢失。

#### 必做

1. **`lib/hfq.ts`**
   - `PtySessionInfo` 增加可选：`shellKind?: string | null` · `alive?: boolean`
   - `HfqBridge` 增加：
     ```ts
     ptyGetScrollback: (payload: { id: string; maxChars?: number }) => Promise<{
       id: string;
       data: string;
       truncated: boolean;
       bytes: number;
       chars: number;
     }>;
     ```

2. **`terminal-store`**
   - 保持：`refresh` 后若 `activeId` 不在 list → 选 `sessions[0]?.id ?? null`（已有，勿回退）。
   - 可选：`getScrollback(id)` 薄封装。
   - workspace 变化：已有 `workspacePath` effect 调 `refresh`；确保 **无 workspace** 时 sessions 清空、activeId null。

3. **`TerminalPanel` 重连策略（推荐最小可用）**

   **A. 全局 data 订阅 + 每会话 scrollback 缓存（推荐）**

   - 模块/组件级：`Map<id, string>` 缓存各会话输出（与 BE ring 互补，覆盖 inactive 时 UI 侧丢失）。
   - **挂载一次**（不依赖 activeId）：
     ```ts
     hfq.onPtyData(({ id, data }) => {
       appendCache(id, data);
       if (id === activeIdRef.current) term.write(data);
     });
     hfq.onPtyExit(({ id, exitCode }) => {
       // 写一行退出提示到 cache + 若 active 则 term.writeln
       void refresh();
     });
     ```
   - **切换 activeId / 初次有 active**：
     ```ts
     term.reset(); // 或 clear
     // 1) 优先本地 cache
     // 2) 若 cache 空或首次 attach：await ptyGetScrollback({ id }) 写入 term + 填 cache
     // 3) fit + ptyResize
     ```
   - 关闭 kill：清该 id cache。
   - workspace 切换：清空全部 cache + refresh。

   **B. 仅 remount 回放（若暂不做多 tab cache）**

   - 至少：active 变化或 mount 时 `ptyGetScrollback` → `term.write`，再订 `onPtyData`。
   - 接受：非 active 会话在 FE 侧看不到后台输出（BE ring 仍在，切回可回放）。

4. **UI 文案**
   - 空态保持中文「无终端会话 / 还没有终端」。
   - tab 标签：`label || shellKind || basename(shell) || id.slice(0,6)`。
   - 错误 toast 短中文（create/kill 失败）。
   - `truncated` 可选在回放前 `term.writeln("[…更早输出已截断]")`（一行即可）。

5. **不要**
   - 不要改 Layout / ActivityBar。
   - 不要在 unmount 时 kill PTY（切页应保留进程）。
   - 不要新依赖。

### P2 · Changes 提交流（B2-3）

路径：`features/changes/ChangesPanel.tsx`

现状已有：空 message toast、空 staged toast、成功 toast、j/k、`askAgentToFix` → `setComposerDraft` + `navigate("/chat")` **不 auto-send**。

#### 收口检查清单（缺啥补啥）

1. **Commit 区可见暂存摘要**
   - 显示「已暂存 N 个」+ 最多 5 个 path 截断列表（可放在 message 输入上方）。
2. **按钮禁用**
   - `message.trim()` 空 **或** staged 数为 0 → Commit 按钮 `disabled`（保留 toast 作双保险）。
3. **成功 toast** 含 sha（若 `gitCommit` 返回 `sha`）否则保持现有文件数摘要。
4. **空态**
   - 无 workspace / 非 git：已有中文 EmptyState — 保持，勿生硬英文。
5. **j/k**
   - `isEditableTarget` 已跳过 input/textarea/contenteditable — 回归即可。
6. **请 Agent 修改**
   - 必须：`setComposerDraft` + `navigate("/chat")`；**禁止** `sendMessage` / auto-send。
   - 无 activeSession：toast「请先打开或创建会话」（已有）。
