## 2. 后端必须完成（packages / main）

### B. compression 真用起来（P0）

**现状**  
- `prefs.modelRoles.compression` → `sessionPrefs` → create/open → `AgentSession.opts.compressionModelRole`  
- `runLoop` 仅调用纯函数 `compactChatMessages`（`packages/agent-core/src/compact.ts`）  
- 注释写 “reserved”

**要求**  
1. 在 `runLoop`（及若 `/compact` 有独立路径则一并）当 `compactChatMessages` 判定需要压缩时：  
   - 若 `compressionModelRole?.provider && model`：对 **head**（被裁掉的旧轮次）拼一份可读 transcript，调用 **一次** `provider.chat` 生成摘要（system note，建议 ≤ 1500–2000 字符）。  
   - 将摘要作为 `role:"system"` 的 `[context compacted · llm]` note，再拼 **tail**（keepRecent）+ 原 system 策略与现有启发式一致。  
   - **失败 / 超时 / 空回复**：静默回退启发式结果，**不阻断**主 turn。  
2. 可观察性：至少在消息列表出现 system note，或 `session.meta`/现有 event 可辨「已 LLM 压缩」（优先 system note，少发明事件类型）。  
3. 扩展 `compact.ts` 时保持 **启发式路径纯函数 + 可测**；LLM 路径可放 `compact.ts` 异步函数或 `loop.ts` 内，但要有 unit test（mock provider）。  
4. Worker + local 行为一致（role 已在 protocol/entry）。

**验收**  
- 无 compression 配置 → 行为与 1.1.5 启发式一致。  
- 有 mock compression provider → 长历史 compact 后 messages 含 LLM 摘要 note。  
- provider 抛错 → 仍 compacted 启发式，turn 成功。

### C. Goal 树 sidecar（P0）

**模式**：对齐 `SessionManager` 的 `*.spawn-attempts.json`（`manager.ts` `load/persist/push` + `delete` unlink）。

**路径**：`%data%/sessions/<sessionId>.goals.json`（`ensureDataDirs().sessions`）

**形状**  
```json
{
  "version": 1,
  "sessionId": "sess_…",
  "goals": [
    {
      "taskId": "goal_…",
      "title": "goal: …",
      "status": "in_progress",
      "kind": "goal",
      "objective": "…",
      "progress": 0,
      "budget": { "maxRounds": 40, "maxToolCalls": 80 },
      "parentTaskId": null,
      "blockedReason": null,
      "acceptance": null,
      "updatedAt": "ISO-8601"
    }
  ]
}
```

**行为**  
1. 每次 `task.updated` 且（`kind==="goal"` **或** `title` 以 `goal:` 开头）→ upsert by `taskId`，cap 例如 100，persist。  
2. 写入点：优先 `AgentSession.emit` 路径或 `SessionManager.onEvent` 统一钩子 — **worker 与 local 都要写到盘**。  
3. `open` / `getSnapshot`：history 重建的 `tasks` **与** sidecar merge；同 `taskId` 取 **较新 `updatedAt`/`at`**。  
4. `SessionManager.delete`：unlink `*.goals.json`（与 spawn-attempts 一起）。  
5. 本版 **不强制** 自动建父子树；若有 `parentTaskId` 则原样持久化（子 goal 可后续扩展）。`/goal` 顶层可继续 `parentTaskId` 空。

**测试**  
- session 测：发 `/goal` → 磁盘出现 goals sidecar → 新 `SessionManager` cold `open` → snapshot.tasks 仍含 goal 字段。  
- delete 清 sidecar。

### E. `read_document`（P1）

**位置**：`packages/tools/src/builtin.ts`（+ 导出 hub）

**契约**  
- 输入：`path`（workspace 相对）、可选 `maxChars`（默认合理上限，如 50_000）  
- `resolveWorkspacePath` — **越界拒绝**  
- 扩展名：  
  - 文本类（`.md/.txt/.json/.ts/.csv/…`）：按 UTF-8 读（可复用 read_file 逻辑）  
  - `.docx`：**无 Pyodide**；ZIP 解压读 `word/document.xml`，去标签抽文本（可用 Node 内置 + 简单 XML 剥标签；或轻量依赖，优先零原生）  
  - `.pdf`：尽力纯 JS 抽文本；失败返回明确 `error`/`ok:false`，不 crash  
  - `.xlsx`：可返回 unsupported 清晰错误，或仅首 sheet 极简预览  
- risk: `low`  
- 注册进默认 tool hub，使 agent 可调  

**Skill**：`skills/bundled/document-read/SKILL.md`  
- description 触发：读 pdf/docx/表格/「打开文档」  
- 指示优先 `read_document`，大文件注意 maxChars  

**测试**：构造最小 docx（zip+xml fixture）或单元测 XML 抽字；path escape 拒绝。
