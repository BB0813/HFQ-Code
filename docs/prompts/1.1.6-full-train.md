## 1. 范围（In）

### A. UI polish（已部分落地，须保留）
- Shell 密度 / surface-1·2·3  
- Coding Profile Header chip **热更新**  
- Mermaid 模块级 init + **streaming 安全**  
- Goal banner 优先 objective  
- Tasks 子会话状态分组  
- Memory / Permissions 空态文案  

### B. F1 收尾 · compression 真用起来（P0 · BE）
- `prefs.modelRoles.compression` 已透传到 session  
- **本版**：`runLoop` / `/compact` 在需要缩上下文时，若配置了 compression role，用其 **LLM 摘要 head**，失败则回退现有 `compactChatMessages` 启发式  
- 发出可观察信号（至少 transcript/system note 或 `statusLine`/event，便于 UI 显示「已压缩」）  

### C. F2 首切片 · Goal 树 sidecar（P0 · BE + FE）
- 仿 `*.spawn-attempts.json`：`%data%/sessions/<sessionId>.goals.json`  
- 在 `task.updated` 且 `kind==="goal"`（或 title 以 `goal:` 开头）时 upsert  
- 字段：`taskId, title, status, objective, progress, budget, parentTaskId, blockedReason, acceptance, updatedAt`  
- `open` / snapshot 合并 sidecar（与 history 重建互补；以较新 `updatedAt` 为准）  
- `delete` session 时清理 sidecar  
- Tasks UI：目标列表支持 **parentTaskId 缩进**（一层即可）  

### D. Compact 可配置（P1 · FE + 已有 IPC）
- Settings：`compactMaxChars` 数字输入（8_000–200_000，与 main `setPrefs` 一致）  
- 说明：压缩模型 id 非空时 **会尝试 LLM 摘要**；空则仅启发式  

### E. 文档读取（P1 · tools + skill）
- 新工具 **`read_document`**（或扩展 `read_file`）：  
  - workspace 内 path-escape 拒绝  
  - `.md/.txt/.json/...` → 现有文本路径  
  - `.docx` → 解压 `word/document.xml` 抽文本（无强制 Pyodide）  
  - `.pdf` → 尽力抽文本（纯 JS 轻量解析或明确失败信息）；不引入原生模块若装不上  
  - `.xlsx` → 可选：首 sheet 文本预览或明确 unsupported  
- bundled skill `skills/bundled/document-read/SKILL.md` 指导何时调用  

### F. Memory 检索增强（P2 · 可降级）
- **优先**：不引入 better-sqlite3 原生编译风险  
- 方案：JSON store 旁路 **倒排索引文件**（`index.v1.json`）双写，search 走 index + 现有 BM25；或 Node 22 `node:sqlite` **可选** FTS5（detect 失败则回退 JSON）  
- 保持 `list`/`export` 仍可读 notes JSON  

### G. 明确本版不做
- 完整 Athena Goal OS / 多租户  
- 替换 Layout A 为自由停靠 IDE  
- FTS 强依赖原生编译失败即红的 CI  
- IM / 邮件 / OCR

## 2. 数据契约（增量）

### goals sidecar
```json
{
  "version": 1,
  "sessionId": "sess_…",
  "goals": [
    {
      "taskId": "goal_abcd1234",
      "title": "goal: …",
      "status": "in_progress",
      "kind": "goal",
      "objective": "…",
      "progress": 0,
      "budget": { "maxRounds": 40, "maxToolCalls": 80 },
      "parentTaskId": null,
      "blockedReason": null,
      "acceptance": null,
      "updatedAt": "ISO"
    }
  ]
}
```

### compact 行为
1. 启发式 `compactChatMessages` 先算是否超预算  
2. 若 `compacted && compressionModelRole`：对 head 调一次 chat 生成 ≤ N 字摘要 system note，再拼 tail  
3. 任一失败 → 仅用启发式结果
