## 1. 一句话定位

**HFQ Code** 是一款 **Windows 桌面端编程智能体**（同类产品：WorkBuddy、ZCode）：

- 打开本地**工作区**
- 跑**智能体会话**（对话、改文件、跑命令）
- 审**改动差异**、管**技能 / 外部工具服务 / 模型**
- 全程带**权限确认**与**操作审计**

**不是：** 即时通讯机器人网关、多租户云端智能体、社交多账号助手。

仓库文件夹名可能仍为 `HFQ_Clod-Agent`，产品名以 **HFQ Code** 为准。

## 2. 产品边界（已冻结）

| 决策项 | 结论 |
|--------|------|
| 产品形态 | 完整图形界面的桌面应用（Electron） |
| 智能体运行时 | TypeScript 自研核心；以后可加 Python 旁路，但不改主壳 |
| 主场景 | **写代码 / 改代码 / 查代码** 优先 |
| 技能兼容 | 读取标准技能说明文件；轻量兼容开放技能元数据；商店渐进开放 |
| 工作区规则 | 可读可写 `AGENTS.md`；可导入 Claude / Cursor 等规则 |
| 配置位置 | 自有用户数据目录（如 `%APPDATA%/HFQ-Code/`） |
| 安全底线 | 禁止路径逃出工作区；密钥不进代码仓库 |
| 更新策略 | **手动**：检查 → 下载 → 用户自己安装；不做静默自动安装（除非以后做代码签名并单独拍板） |
| 界面语言 | 产品界面中文；技术文档可中英混排 |

大改动前仍应对照：`docs/DECISIONS.md`、`docs/ARCHITECTURE.md`。

## 4. 已交付功能（按用户能感知到的）

### 4.1 会话与智能体

- 新建 / 恢复 / 停止 / 删除 / 重命名会话；自动生成标题  
- 流式对话、用量统计、长对话压缩、**计划模式**、**子智能体**  
- 会话跑在**子进程**里，崩溃有失败事件  
- 斜杠命令：例如 `/goal` 长目标（提高本轮轮次与工具次数）、`/compact` 等  

### 4.2 工具能力（智能体能调用的）

读文件、列目录、搜索、Git 状态/差异/查看/提交、记忆读写、写文件、打补丁、执行命令、网络请求、拉起子智能体，以及外部工具服务注入的动态工具。

### 4.3 权限与治理

- 四种访问模式：  
  - **变更前确认**（默认）  
  - **自动编辑**（写文件类少问，危险命令仍问）  
  - **计划模式**（禁止真正改环境的操作）  
  - **完全访问**（几乎全自动，含危险命令，界面有警告）  
- 全局默认 + **当前会话可单独切换**  
- 权限弹窗支持排队；只有授权成功才关闭，避免卡住  
- **约 10 分钟无操作自动拒绝**  
- 会话失败 / 用户停止 / 子进程崩溃：**清空待授权并解锁输入框**  
- 提交代码类操作会在授权摘要里突出**提交说明**  
- 权限矩阵页、本会话白名单、审计时间线  

### 4.4 技能与「技能商店」

| 能力 | 大约版本 |
|------|----------|
| 离线索引导航 + 可选远程目录文件 | 1.0.5 |
| 从本地文件夹安装技能 | 1.0.5 |
| 技能说明预览抽屉、重名覆盖、标签筛选 | 1.0.6 |
| **远程 https 压缩包安装**（体积上限、超时、可选校验和、防路径逃逸、**不执行包内脚本**） | **1.0.9** |

**政策：** 远程包只当「数据 + 技能说明」，不是任意安装程序。

### 4.5 更新与安装分发

- 应用内「检查更新」：优先镜像，失败则备用镜像 → 中转接口 → 直连官方接口  
- 只提示 / 打开下载页，**不会偷偷安装**  
- 打标签后自动打 Windows 安装包与绿色版，并生成校验文件  
- **未做发布者数字签名** → 系统可能提示「未知发布者」；`README` 已写如何处理  

### 4.6 其它已交付

- 密钥与主配置分离存放  
- 记忆（用户 / 项目范围）、用量面板、从其它产品导入规则  
- 外部工具：标准输入输出进程 + 网络接口两种  
- 对话区顶栏固定 **提供方 · 模型**；输入区模型按钮只显示模型名；访问/模型菜单**向上弹出**

| 「完全访问」过宽 | 危险命令也可能不问 | 界面警告；默认仍是「变更前确认」 |
| 远程技能包供应链 | 恶意压缩包 | 仅加密网站、体积与超时、校验和失败则拒、禁路径逃逸、不跑脚本 |

## 附录 B：安装包与系统提示（摘要）

当前正式构建使用**自签**发布者 **HFQ-ClodBreeze**（非商业 EV 证书）。用户仍可能看到 SmartScreen / 系统保护提示，属预期。  

**推荐做法：** 只从本仓库发布页下载 → 可选核对校验文件 → 选择「运行」→ 若有系统保护则「更多信息 → 仍要运行」。NSIS 安装会尝试静默导入发布者信任。  

**自签做不到的事：** 让系统长期完全信任并彻底去掉 SmartScreen——通常需要付费代码签名证书与下载声誉。  

详情见 [PACKAGING.md](./PACKAGING.md) 与仓库根目录 `README.md`。
