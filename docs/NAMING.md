# Naming Recommendation (Q4)

## Recommendation

| Field | Value |
|-------|--------|
| **Product name** | **HFQ Code** |
| **Short name** | HFQ |
| **Package / repo** | `hfq-code` |
| **App user model id** | `club.binbim.hfqcode` |
| **Data dir (Windows)** | `%APPDATA%/HFQ-Code/` |
| **CLI (optional later)** | `hfq` |
| **Window title** | HFQ Code |

### Why this name

1. Keeps your **HFQ** brand; easy to say and search in Chinese/English contexts.  
2. **Code** states the primary scenario (coding agent), unlike “Clod/Claw Agent” which sounds like a generic chatbot or OpenClaw clone.  
3. Aligns with category peers: WorkBuddy, ZCode, Cursor — short product noun + coding signal.  
4. Leaves room for future lines: HFQ Code (desktop), HFQ CLI, HFQ Brain.  
5. Avoids trademark-adjacent “Claude / Codex / Claw” collisions while still allowing OpenClaw-compatible skill loading under the hood.
