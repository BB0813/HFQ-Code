方案 D · 三合一怎么拼

来自 A · Cursor

  • 左侧活动栏 + 会话列表
  • 右侧改动检视（主审阅面）
  • 对话是默认中心，不是附属标签

来自 B · VS Code

  • 可折叠底栏终端（PTY 首选挂载）
  • 活动栏可切「文件」显示轻量树
  • 底栏多 tab：终端 / 输出 / 问题

来自 C · Claude/Zed

  • 中央时间线干净、少 chrome
  • 顶栏极简：工作区 / 标题 / 关键开关
  • 次要入口收进「更多」与 / 菜单

默认布局与折叠规则

  • 默认开：活动栏 + 会话列表 + 对话 + 右侧改动（有 diff 时）
  • 默认收：底栏终端（点「终端」或 Agent 引导再展开）
  • 文件树：按需，不默认占主路径
  • 窄屏：先藏右侧，再藏侧栏；底栏全宽

框架建议（若定 D）

与 A/B 相同：生产壳倾向 React 或 Preact + 现有 CSS tokens； Agent 核心保持 TS monorepo。可先 vanilla 分区验证再迁框架。

审核回复示例：「定 D」/「定 D，但默认关掉右侧」/「定 D，文件树不要」
